\section{Giới thiệu}

\subsection{Bối cảnh hình thành đề tài}
Chuyển đổi số trong cơ quan hành chính cấp cơ sở là nhu cầu cấp thiết nhằm nâng cao chất lượng phục vụ người dân, giảm thời gian xử lý hồ sơ và minh bạch thông tin. Trong thực tế vận hành tại cấp xã, các nhu cầu thường xuyên bao gồm:
\begin{itemize}[leftmargin=1.1cm]
  \item Công bố thông báo, tin tức địa phương theo danh mục và theo thời gian.
  \item Cung cấp danh mục dịch vụ công và hướng dẫn hồ sơ, lệ phí, thời gian xử lý.
  \item Tiếp nhận yêu cầu trực tuyến, theo dõi trạng thái hồ sơ và phản hồi công dân.
  \item Quản trị nội dung tập trung với cơ chế phân quyền rõ ràng giữa các vai trò cán bộ.
\end{itemize}

Dự án \textbf{XaTanThuanDong} ra đời để giải quyết trọn vẹn các nhu cầu trên theo định hướng triển khai thực tế tại địa phương. Hệ thống không chỉ dừng ở website thông tin một chiều mà còn mở rộng thành nền tảng tương tác hai chiều giữa người dân và chính quyền cấp xã.

\subsection{Bài toán đặt ra}
Bài toán nghiệp vụ tổng thể có thể mô tả như sau:
\begin{enumerate}[leftmargin=1.1cm]
  \item Xây dựng cổng thông tin công khai, dễ tra cứu, thân thiện trên cả thiết bị di động và máy tính.
  \item Xây dựng khu vực quản trị để tạo/sửa/xóa nội dung, quản lý danh mục, duyệt bình luận, quản lý media và xử lý hồ sơ dịch vụ.
  \item Đảm bảo an toàn ở mức ứng dụng web hiện đại: xác thực, phân quyền, kiểm tra dữ liệu đầu vào, ghi log và xử lý lỗi tập trung.
  \item Đảm bảo khả năng mở rộng trong tương lai: thêm thủ tục, thêm chuyên mục, tích hợp cổng dịch vụ công cấp trên.
\end{enumerate}

\subsection{Mục tiêu của hệ thống}
Mục tiêu tổng quát của đồ án là thiết kế và hiện thực một hệ thống web đầy đủ ba lớp gồm giao diện người dùng, tầng dịch vụ API và tầng dữ liệu. Từ mục tiêu tổng quát, hệ thống cụ thể hóa thành các mục tiêu sau:
\begin{itemize}[leftmargin=1.1cm]
  \item \textbf{Mục tiêu chức năng}: cung cấp đầy đủ nhóm chức năng công khai và chức năng quản trị theo vai trò.
  \item \textbf{Mục tiêu kỹ thuật}: áp dụng .NET 8 Web API, Entity Framework Core, Identity, JWT và React theo chuẩn hiện đại.
  \item \textbf{Mục tiêu vận hành}: khởi tạo dữ liệu mẫu tự động, hỗ trợ chạy môi trường phát triển nhanh, có tài liệu endpoint và có khả năng mở rộng triển khai production.
  \item \textbf{Mục tiêu chất lượng}: phản hồi API rõ ràng, lỗi được chuẩn hóa, giao diện phản hồi tốt trên nhiều kích thước màn hình.
\end{itemize}

\subsection{Phạm vi triển khai}
Phạm vi của báo cáo và hệ thống bao gồm:
\begin{itemize}[leftmargin=1.1cm]
  \item \textbf{Backend}: dự án \texttt{backend/XaTanThuanDong.Api}, triển khai API cho public và admin, có middleware xử lý lỗi, có seed dữ liệu ban đầu.
  \item \textbf{Frontend người dân}: dự án \texttt{frontend/xatanthuandong-web}, cung cấp trang chủ, tin tức, dịch vụ, thư viện, liên hệ và một phần giao diện admin rút gọn.
  \item \textbf{Frontend portal quản trị}: dự án \texttt{frontend/xatanthuandong-portal}, tập trung vào các màn hình CRUD quản trị chuyên sâu.
  \item \textbf{Cơ sở dữ liệu}: SQL Server với lược đồ do EF Core migrations quản lý.
\end{itemize}

Ngoài phạm vi hiện tại, các tích hợp như chữ ký số, thanh toán trực tuyến, đồng bộ một cửa liên thông hoặc tích hợp định danh điện tử mức sâu chưa được triển khai trong mã nguồn hiện tại.

\subsection{Phương pháp thực hiện báo cáo}
Báo cáo này được xây dựng theo phương pháp \textbf{code-driven documentation}, tức là:
\begin{enumerate}[leftmargin=1.1cm]
  \item Đọc trực tiếp mã nguồn của toàn bộ module backend và frontend.
  \item Đối chiếu controllers, services, models, validators, cấu hình runtime và migration để tái dựng thiết kế hệ thống thực tế.
  \item Trích xuất endpoint và luồng nghiệp vụ từ code thay vì mô tả giả định.
  \item Tổng hợp test case và tiêu chí đánh giá dựa trên hành vi ứng dụng đã cài đặt.
\end{enumerate}

\subsection{Giá trị thực tiễn}
Hệ thống mang lại các lợi ích thực tiễn:
\begin{itemize}[leftmargin=1.1cm]
  \item Người dân tiếp cận thông tin nhanh, tra cứu thủ tục minh bạch và gửi hồ sơ trực tuyến thuận tiện.
  \item Cán bộ xã có công cụ quản trị tập trung, giảm thao tác thủ công và tăng khả năng theo dõi tiến độ.
  \item Dữ liệu nội dung và dịch vụ được chuẩn hóa, giúp công tác vận hành nhất quán hơn.
  \item Nền tảng kỹ thuật mở rộng tốt để phát triển các tính năng nâng cao ở giai đoạn tiếp theo.
\end{itemize}

\section{Cơ sở lý thuyết}

\subsection{Kiến trúc ba lớp trong ứng dụng web}
Kiến trúc tổng thể của hệ thống tuân theo mô hình ba lớp:
\begin{itemize}[leftmargin=1.1cm]
  \item \textbf{Presentation Layer}: giao diện React ở hai dự án frontend.
  \item \textbf{Application/API Layer}: ASP.NET Core Web API xử lý nghiệp vụ, xác thực, phân quyền.
  \item \textbf{Data Layer}: SQL Server thông qua Entity Framework Core.
\end{itemize}

Mô hình này giúp phân tách trách nhiệm rõ ràng, thuận lợi cho bảo trì, mở rộng và kiểm thử từng phần độc lập.

\subsection{RESTful API và chuẩn trao đổi dữ liệu}
Hệ thống sử dụng RESTful API với định dạng dữ liệu JSON. Các nguyên tắc được áp dụng trong code:
\begin{itemize}[leftmargin=1.1cm]
  \item Đường dẫn tài nguyên theo danh từ số nhiều: \texttt{/api/public/articles}, \texttt{/api/admin/users}.
  \item Tách route public và route admin theo namespace rõ ràng.
  \item HTTP methods phản ánh đúng hành vi: GET (đọc), POST (tạo), PUT (cập nhật), DELETE (xóa).
  \item Trả về mã trạng thái phù hợp: 200, 202, 204, 400, 401, 404, 500.
\end{itemize}

\subsection{Xác thực và phân quyền bằng JWT}
Ứng dụng triển khai JWT Bearer Authentication trong \texttt{Program.cs}. Token được tạo tại \texttt{TokenService} với các đặc điểm:
\begin{itemize}[leftmargin=1.1cm]
  \item Claims chuẩn: \texttt{sub}, \texttt{email}, \texttt{nameidentifier}, \texttt{name}.
  \item Claims vai trò: thêm danh sách role của người dùng để phục vụ \texttt{[Authorize(Roles=...)]}.
  \item Thuật toán ký: HMAC SHA256.
  \item Thời hạn token: 8 giờ.
\end{itemize}

Phân quyền được triển khai theo role-based access control:
\begin{itemize}[leftmargin=1.1cm]
  \item \texttt{Admin}: toàn quyền quản trị, bao gồm quản lý người dùng.
  \item \texttt{Editor}: quản trị nội dung/dịch vụ theo phạm vi cho phép.
  \item \texttt{Viewer}: xem dashboard tổng hợp.
\end{itemize}

\subsection{ASP.NET Core Identity}
Identity cung cấp cơ chế quản lý tài khoản, mật khẩu, role, claims và các bảng hệ thống chuẩn. Ưu điểm của lựa chọn này:
\begin{itemize}[leftmargin=1.1cm]
  \item Giảm rủi ro khi tự viết cơ chế bảo mật từ đầu.
  \item Khả năng mở rộng tốt cho các yêu cầu reset mật khẩu, xác thực hai lớp trong tương lai.
  \item Tích hợp trực tiếp với EF Core và DI container của ASP.NET Core.
\end{itemize}

Trong cấu hình hiện tại, chính sách mật khẩu yêu cầu tối thiểu 6 ký tự và có chữ số. Email phải duy nhất.

\subsection{Entity Framework Core và Migration}
EF Core được dùng để ánh xạ model C\# sang bảng dữ liệu SQL Server. Ưu điểm chính:
\begin{itemize}[leftmargin=1.1cm]
  \item Khai báo ràng buộc dữ liệu ở tầng model để đồng bộ giữa code và DB.
  \item Quản lý thay đổi lược đồ bằng migration có phiên bản.
  \item Hỗ trợ truy vấn LINQ mạnh và tối ưu hóa tương đối tốt với chỉ mục phù hợp.
\end{itemize}

Ứng dụng sử dụng cơ chế tự động migrate khi khởi động, giúp môi trường chạy mới được khởi tạo DB nhanh.

\subsection{Validation với FluentValidation}
Thay vì kiểm tra rải rác trong controller, dự án áp dụng FluentValidation cho các DTO nhập liệu. Mỗi request quan trọng đều có validator tương ứng:
\begin{itemize}[leftmargin=1.1cm]
  \item LoginRequestValidator.
  \item UpsertCategoryRequestValidator.
  \item UpsertArticleRequestValidator.
  \item CreateCommentRequestValidator.
  \item UpsertServiceProcedureRequestValidator.
  \item CreateApplicationRequestValidator.
  \item UpdateApplicationStatusRequestValidator.
\end{itemize}

Lợi ích là thông báo lỗi thống nhất, giảm bug đầu vào và nâng cao độ tin cậy dữ liệu.

\subsection{Xử lý lỗi tập trung bằng middleware}
\texttt{GlobalExceptionMiddleware} bắt các lỗi chưa xử lý và trả về dạng \texttt{application/problem+json}. Mỗi lỗi có:
\begin{itemize}[leftmargin=1.1cm]
  \item Mã trạng thái tương ứng theo loại exception.
  \item \texttt{traceId} để truy vết log.
  \item \texttt{detail} chỉ hiển thị trong môi trường Development.
\end{itemize}

Đây là cách tiếp cận đúng chuẩn cho API production vì tránh lộ stack trace và tăng khả năng theo dõi vận hành.

\subsection{Logging với Serilog}
Hệ thống dùng Serilog với hai sink chính: Console và File rolling theo ngày. Cấu hình này đem lại:
\begin{itemize}[leftmargin=1.1cm]
  \item Quan sát log thời gian thực ở môi trường phát triển.
  \item Lưu vết lịch sử vận hành để hậu kiểm và phân tích sự cố.
  \item Khả năng mở rộng thêm sink sang Elastic, Seq hoặc cloud logging.
\end{itemize}

\subsection{Khái niệm SPA và React Router}
Frontend dùng React theo mô hình SPA:
\begin{itemize}[leftmargin=1.1cm]
  \item Điều hướng client-side bằng React Router.
  \item Tái sử dụng layout với \texttt{Outlet} cho từng nhóm route.
  \item Tách lớp gọi API bằng Axios instance trong \texttt{lib/api.js}.
\end{itemize}

Trong dự án có hai ứng dụng frontend độc lập:
\begin{itemize}[leftmargin=1.1cm]
  \item \texttt{xatanthuandong-web}: cổng người dân và một phần admin rút gọn.
  \item \texttt{xatanthuandong-portal}: portal admin chuyên sâu.
\end{itemize}

\subsection{Nguyên lý bảo mật web áp dụng}
Từ mã nguồn hiện tại có thể thấy các nguyên lý đã được áp dụng ở mức cơ bản:
\begin{itemize}[leftmargin=1.1cm]
  \item Xác thực dựa trên token và kiểm tra role ở endpoint admin.
  \item Validate dữ liệu phía server.
  \item Giới hạn loại file upload theo đuôi và MIME.
  \item Tách API public và API admin, thêm cơ chế ẩn admin route ở môi trường Production.
\end{itemize}

\subsection{Khái niệm kiểm thử trong hệ thống web}
Kiểm thử trong đồ án này bao gồm các lớp:
\begin{itemize}[leftmargin=1.1cm]
  \item Kiểm thử chức năng API theo endpoint.
  \item Kiểm thử tích hợp frontend-backend theo luồng nghiệp vụ.
  \item Kiểm thử đầu vào lỗi và kiểm tra thông báo phản hồi.
  \item Kiểm thử tương thích giao diện và tính phản hồi.
\end{itemize}

\section{Phân tích thiết kế}

\subsection{Khảo sát yêu cầu chức năng}
Từ mã nguồn đã triển khai, có thể nhóm yêu cầu chức năng thành hai nhóm lớn: công khai và quản trị.

\subsubsection{Nhóm công khai dành cho người dân}
\begin{itemize}[leftmargin=1.1cm]
  \item Xem danh mục tin tức hoạt động.
  \item Xem danh sách bài viết đã xuất bản, lọc theo danh mục, tìm kiếm theo từ khóa.
  \item Xem chi tiết bài viết theo slug.
  \item Gửi bình luận cho bài viết, chờ duyệt.
  \item Xem danh sách thủ tục hành chính đang hoạt động.
  \item Nộp hồ sơ thủ tục trực tuyến và nhận mã hồ sơ.
  \item Tra cứu trạng thái hồ sơ bằng mã.
  \item Xem danh sách lãnh đạo/cán bộ theo nhóm.
  \item Xem thư viện media có phân trang.
\end{itemize}

\subsubsection{Nhóm quản trị nội bộ}
\begin{itemize}[leftmargin=1.1cm]
  \item Đăng nhập quản trị và nhận token JWT.
  \item Xem dashboard tổng hợp số liệu.
  \item Quản lý danh mục tin tức.
  \item Quản lý bài viết (CRUD).
  \item Duyệt/xóa bình luận.
  \item Quản lý thông tin lãnh đạo và ảnh đại diện.
  \item Quản lý media: upload, thay ảnh, cập nhật topic/title, xóa.
  \item Quản lý thủ tục hành chính.
  \item Quản lý hồ sơ dịch vụ và cập nhật trạng thái.
  \item Quản lý tài khoản người dùng và phân quyền role.
\end{itemize}

\subsection{Cài đặt backend chi tiết}

\subsubsection{Đăng ký dịch vụ trong Program}
\begin{lstlisting}[style=appcode]

builder.Services.AddValidatorsFromAssemblyContaining<Program>();

Phần cài đặt trên đảm bảo ứng dụng có đầy đủ pipeline cho API bảo mật, kiểm tra dữ liệu và truy cập cơ sở dữ liệu.

\subsubsection{Pipeline middleware}
Trình tự middleware chính trong code:
\begin{enumerate}[leftmargin=1.1cm]
  \item Serilog request logging.
  \item GlobalExceptionMiddleware.
  \item Swagger (chỉ development).
  \item HTTPS redirection.
  \item CORS policy cho frontend.
  \item Cơ chế ẩn admin API trong Production.
  \item Routing + Authentication + Authorization.
  \item MapControllers.
  \item Static files và static files cho thư mục Uploads.
\end{enumerate}

\subsubsection{DbContext và ràng buộc dữ liệu}
Tại \texttt{AppDbContext}, các ràng buộc được khai báo rõ:
\begin{itemize}[leftmargin=1.1cm]
  \item Độ dài tối đa cho tên, slug, email, trạng thái.
  \item Unique index cho slug danh mục và slug bài viết.
  \item Foreign key với hành vi xóa hợp lý (Restrict hoặc Cascade).
  \item Composite index cho các truy vấn lọc phổ biến.
\end{itemize}

\subsubsection{Seed dữ liệu}
\texttt{SeedData} khởi tạo:
\begin{itemize}[leftmargin=1.1cm]
  \item Role mặc định: Admin, Editor, Viewer.
  \item Tài khoản admin mặc định để vận hành thử nghiệm.
  \item Danh mục mẫu, thủ tục mẫu, bài viết mẫu.
  \item Dữ liệu được thiết kế idempotent để tránh trùng khi chạy nhiều lần.
\end{itemize}

\section*{Phụ lục E: Kịch bản demo vận hành}
\addcontentsline{toc}{section}{Phụ lục E: Kịch bản demo vận hành}

\subsection*{E.1 Kịch bản 1: Quản trị đăng bài và người dân đọc tin}
\begin{enumerate}[leftmargin=1.1cm]
  \item Admin đăng nhập portal bằng tài khoản hợp lệ.
  \item Vào màn hình quản lý danh mục, tạo danh mục mới nếu cần.
  \item Vào màn hình tạo bài viết, nhập tiêu đề, sinh slug, nhập nội dung, chọn status Published.
  \item Upload ảnh đại diện, lưu bài.
  \item Mở frontend công khai, vào mục Tin tức và xác nhận bài đã hiển thị.
  \item Mở chi tiết bài bằng slug và xác nhận nội dung/ảnh đúng như dữ liệu đã nhập.
\end{enumerate}

\subsection*{E.2 Kịch bản 2: Người dân nộp hồ sơ trực tuyến}
\begin{enumerate}[leftmargin=1.1cm]
  \item Truy cập trang Dịch vụ hành chính trên frontend công khai.
  \item Chọn thủ tục, nhập đầy đủ thông tin họ tên, email, số điện thoại, địa chỉ.
  \item Gửi hồ sơ và nhận mã hồ sơ do hệ thống trả về.
  \item Dùng mã hồ sơ để tra cứu trạng thái ngay trên giao diện dịch vụ.
  \item Admin đăng nhập portal, vào màn hình Hồ sơ dịch vụ để xác nhận hồ sơ mới đã xuất hiện.
  \item Admin cập nhật trạng thái hồ sơ sang Processing hoặc Approved.
  \item Người dân tra cứu lại và thấy trạng thái đã được cập nhật.
\end{enumerate}

\subsection*{E.3 Kịch bản 3: Duyệt bình luận và hiển thị công khai}
\begin{enumerate}[leftmargin=1.1cm]
  \item Người dân vào chi tiết bài viết và gửi bình luận.
  \item Kiểm tra API public comments: bình luận mới chưa xuất hiện do đang chờ duyệt.
  \item Admin vào màn hình bình luận, duyệt bình luận vừa gửi.
  \item Quay lại API public comments và giao diện chi tiết bài viết để kiểm tra bình luận đã hiển thị.
\end{enumerate}

\subsection*{E.4 Kịch bản 4: Upload media và hiển thị thư viện}
\begin{enumerate}[leftmargin=1.1cm]
  \item Admin vào màn hình media, chọn ảnh hợp lệ và upload.
  \item Cập nhật tiêu đề/topic cho media vừa upload.
  \item Mở trang Thư viện ở frontend công khai, kiểm tra ảnh xuất hiện đúng với metadata.
  \item Thực hiện phân trang qua nhiều trang ảnh, đảm bảo không lỗi hiển thị.
\end{enumerate}

\subsection*{E.5 Kịch bản 5: Quản lý người dùng và phân quyền}
\begin{enumerate}[leftmargin=1.1cm]
  \item Admin tạo tài khoản mới role Viewer.
  \item Đăng nhập bằng tài khoản Viewer và thử truy cập endpoint users admin.
  \item Xác nhận endpoint users trả 403 do thiếu quyền Admin.
  \item Đổi role tài khoản thành Editor rồi kiểm tra lại quyền trên các endpoint content/services.
  \item Xác nhận hành vi quyền truy cập đúng theo định nghĩa role.
\end{enumerate}
